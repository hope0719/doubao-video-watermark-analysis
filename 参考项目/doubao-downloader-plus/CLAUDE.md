## Project Overview

doubao-downloader (豆包下载器) is a browser extension and userscript for downloading watermark-free AI-generated images and videos from Doubao (豆包, `www.doubao.com/chat/*`). The same codebase produces both a Tampermonkey/Greasemonkey userscript and a Chrome Extension (Manifest V3).

## Commands

- **Dev server:** `pnpm dev` — starts Vite dev server with userscript hot-reload via vite-plugin-monkey
- **Build:** `pnpm build` — runs `tsc` (type-check) then `vite build` (produces `dist/doubao-downloader.user.js` and extension files)
- **Preview:** `pnpm preview` — preview production build locally

There are no test or lint scripts. Code quality is enforced via TypeScript strict mode (`tsc` in the build step).

## Architecture

### Data Interception

The core mechanism is a global `JSON.parse` monkey-patch (`src/hooks/use-json.ts`). It intercepts all JSON responses from Doubao's API in the page's MAIN world, extracting watermark-free image/video URLs from `creation` data (`image_ori_raw.url`). This is what makes the extension work without any network request modification.

### Data Flow

1. `useJson` hook patches `JSON.parse` → extracts `ConvMessage[]` from API responses
2. `ConvContext` (React context) holds the intercepted conversation data
3. `ConvFilterContext` manages filtering/pagination state
4. `useConvs` / `useCreationPagination` hooks compute derived state
5. `ImageList` / `ImageCard` render the image/video grid
6. `useDownload` handles downloads: single files via `file-saver`, batch downloads via streaming ZIP (`streamsaver` + custom `src/lib/zip-stream.js`)
7. Settings and download history persist in IndexedDB via Dexie (`src/db/index.ts`)

### Key Directories

- `src/components/` — UI components (Semi UI + Tailwind CSS with `dd:` prefix)
- `src/context/` — React contexts (ConvContext, ConvFilterContext, SettingContext)
- `src/hooks/` — Custom hooks including the JSON interception, download logic, and settings persistence
- `src/lib/` — Utilities including a custom streaming ZIP writer (`zip-stream.js`)
- `src/api/` — Direct API calls to Doubao (currently just video URL fetching)

### Build Configuration

- **Vite 7** with `@vitejs/plugin-react`, `@tailwindcss/vite`, `vite-plugin-monkey`, and `vite-plugin-static-copy`
- Path alias: `@` → `./src` (configured in both `vite.config.ts` and `tsconfig.json`)
- Build-time defines: `__APP_VERSION__` and `__BUILD_TIME__` (injected from package.json)
- Tailwind CSS 4 uses a `dd:` prefix to avoid class conflicts with Doubao's own styles
- TypeScript strict mode with `noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch`

### Dual Build Outputs

- **Userscript:** `dist/doubao-downloader.user.js` — generated by vite-plugin-monkey, targets `https://www.doubao.com/chat/*`
- **Chrome Extension:** `dist/manifest.json` (version injected at build time) + `dist/popup.html` + `dist/logo.png` + bundled JS
