> 生成日期：2026-07-05
> 状态：已完成调研
> 调研范围：AI去水印工具、开源项目源码分析、小程序去水印原理、图片vs视频水印差异

## 一、核心发现：图片水印 vs 视频水印的本质区别

经过深入调研，发现豆包的图片水印和视频水印有**根本性的技术差异**：

| 维度 | 图片水印 | 视频水印 |
|------|---------|---------|
| 水印类型 | 前端叠加层（CSS/Canvas Overlay） | 服务端编码嵌入（像素级烘焙） |
| 添加时机 | 浏览器渲染时叠加 | 视频编码阶段写入 |
| 可去除性 | ✅ 可通过提取原始URL绕过 | ❌ 无法通过URL/API绕过 |
| doubao-nomark效果 | ✅ 有效（提取原始图片URL） | ❌ 无效（API返回的就是有水印的） |
| 第三方工具效果 | ✅ 多种方案可用 | ⚠️ 只能靠AI修复或裁剪 |

### 1.1 图片水印为什么可以绕过

豆包网页版生成图片时：
1. 服务器返回**无水印的原始图片URL**（嵌入在页面SSR数据中）
2. 浏览器前端通过CSS/Canvas在水印层叠加"AI生成"标识
3. 用户右键保存时，保存的是**叠加水印后的版本**

因此，只要能提取到原始图片URL，就能获得无水印图片。doubao-nomark 的 `image.py` 正是通过解析页面HTML中的 JSON 数据，直接提取 `image_ori_raw.url` 字段获取无水印原图。

### 1.2 视频水印为什么无法绕过

豆包视频在服务端编码时：
1. 视频生成完成后，编码器在画面中烘焙水印
2. CDN只存储**一份带水印的视频文件**
3. 无论用什么API、什么参数、什么UA，返回的都是同一份文件
4. `lr=unwatermarked` 和 `lr=video_gen_watermark_dyn` 返回的文件**MD5完全一致**

## 二、已调研的替代方案

### 方案A：Morph Studio — AI视频去水印（后处理）

| 维度 | 详情 |
|------|------|
| 官网 | https://www.morphstudio.com/video-watermark-remover |
| 类型 | AI逐帧修复（AI Inpainting） |
| 原理 | 逐帧分析视频 → AI检测水印区域 → 用周围像素重建背景 → 输出无水印帧 |
| 支持水印 | 静态水印、动态水印（Sora/Kling/Runway/TikTok等） |
| 使用方式 | 上传视频 → 用画笔标记水印区域 → AI处理 → 下载 |
| 免费额度 | 有限免费（视频<10分钟、<50MB） |
| 画质 | 无模糊，AI重建区域自然 |
| 关键限制 | **不是获取原始无水印视频，而是AI重建水印区域** |

**评估**：
- ✅ 可以去除任何位置的水印（包括画面中央）
- ✅ 支持动态水印
- ❌ 需要手动标记水印区域
- ❌ 处理速度取决于视频长度和AI模型
- ❌ 重建区域不是原始像素（虽然视觉效果好）
- ❌ 免费额度有限

### 方案B：doubao-nomark 源码深度分析

| 维度 | 详情 |
|------|------|
| 仓库 | https://github.com/ihmily/doubao-nomark |
| 版本 | v1.0.5（2026-05-22 更新） |
| 图片提取 | ✅ 有效（通过解析页面SSR JSON数据获取原始图片URL） |
| 视频提取 | ❌ 无效（使用与我们相同的API和参数） |

**视频提取代码分析**（`doubao_parser/video.py`）：

```python
# 与我们的测试完全相同的调用方式
response = await client.post(
    "https://www.doubao.com/samantha/media/get_play_info",
    params=params,           # version_code, aid, real_aid, samantha_web, use-olympus-account...
    headers=headers,         # 微信PC版UA
    json={"key": vid},       # video_id
)
result["data"]["original_media_info"]["main_url"]  # 返回有水印的URL
```

**关键结论**：doubao-nomark 的视频提取使用与标准方法完全相同的 API 调用，返回的文件 MD5 与我们之前测试完全一致（`d903cd06...`），视频画面**仍然有水印**。

**v1.0.5 "修复视频提取" 的实际内容**：
- 提取 vid 的正则表达式更新（支持 `/thread/` 链接格式）
- 并非获取了不同的（无水印）视频 URL
- 核心API调用逻辑未改变

### 方案C：doubao-seedance-15s — logo_type 参数移除

| 维度 | 详情 |
|------|------|
| 仓库 | https://github.com/yht-cdd/doubao-seedance-15s |
| 类型 | Edge扩展（Manifest V3） |
| 原理 | 使用 `declarativeNetRequest` 静态规则拦截请求，移除 `logo_type` 参数 |
| 实测结果 | ❌ 无效（CDN缓存key不含查询参数） |

**规则分析**（`src/rules.json`）：

3条规则：
1. 图片URL替换水印模板ID
2. 封面URL替换
3. 视频URL移除 `logo_type=video_gen_watermark_dyn` 参数

**实测验证**：与 `lr` 参数一样，`logo_type` 参数在 CDN 层面同样无效，因为：
- CDN 缓存 key（`X-Kfc-Cachekey`）只包含路径，不含查询参数
- 有无 `logo_type` 参数返回的文件 MD5 完全一致

### 方案D：lsj-watermark-remover — 反向Alpha混合算法

| 维度 | 详情 |
|------|------|
| 仓库 | https://github.com/RusianHu/lsj-watermark-remover |
| 类型 | 纯前端工具（HTML5 Canvas） |
| 适用对象 | Gemini、Doubao（豆包）、Qwen 生成的**图片**水印 |
| 算法 | 反向Alpha混合（Reverse Alpha Blending） |
| 支持视频 | ❌ 不支持 |

**工作原理**：

```
用户输入图片 → 水印特征检测 → 加载基准背景底图
→ 计算Alpha通道映射 → 反向混合运算 → 还原原始RGB值 → 输出无水印图片
```

- 水印是半透明叠加层（`output = alpha * watermark + (1-alpha) * original`）
- 已知 `output`、`watermark`（预存基准底图）和 `alpha`（水印透明度）
- 可通过反向计算还原 `original`
- **仅适用于图片**，视频水印是烘焙在编码中的，不是半透明叠加层

### 方案E：豆包"变清晰"功能（官方隐藏技巧）

| 维度 | 详情 |
|------|------|
| 类型 | 豆包APP内置功能 |
| 适用对象 | 图片（非视频） |

## 三、第三方去水印小程序/工具分析

### 市面上的工具分类

通过搜索发现大量"豆包去水印"文章和工具，主要分为以下几类：

#### 3.1 微信小程序类

常见的小程序包括：奈斯水印助手、水印魔术手、耶斯去水印、大佬去水印等。

**宣传特点**：
- 声称"0.3秒处理30秒视频"
- 声称"本地处理不上传服务器"
- 声称"完全免费无限制"

**技术分析**：
- "0.3秒处理" 对AI逐帧修复来说**不可能实现**（AI修复至少需要几秒/帧）
- "本地处理" 可能指下载过程在本地，但视频源仍来自服务器API
- 这些小程序很可能也是调用 `/samantha/media/get_play_info` API，返回的视频**仍有水印**
- 部分小程序可能在服务器端做AI修复，但宣称"本地处理"以打消隐私顾虑
- **不排除部分小程序在客户端做画面裁剪**（速度快、实现简单）

#### 3.2 桌面软件类

| 工具 | 类型 | 原理 | 评价 |
|------|------|------|------|
| HitPawRemover | 桌面端 | AI自动检测+修复 | 付费，效果取决于AI模型 |
| UnscreenVideo | 在线/桌面 | AI填充 | 免费额度有限 |
| KapwingTool | 在线编辑 | 遮罩+填充 | 免费版自带水印 |
| 剪映 | 桌面端 | 画中画+蒙版 | 手动操作，灵活 |
| FFmpeg | 命令行 | 裁剪 | 快速但损失像素 |

#### 3.3 关键结论

市面上的"豆包去水印"工具：
1. **图片去水印**：多数有效（提取原始URL或Alpha混合算法）
2. **视频去水印**：要么仍然有水印（API方式），要么是AI修复/裁剪（后处理方式）
3. **没有发现任何工具能通过API获取真正的无水印视频**

## 四、bboo.boo 说明

经调研确认，bboo.boo **不是去水印工具**，而是一个代理/线路检测服务（类似"一元机场"）。页面显示"正在自动匹配最佳线路访问官网"，用于访问可能被限制的国外网站（如 Morph Studio、OpenAI 等）。可作为访问国际AI工具的代理使用，但与去水印无直接关系。

## 五、可行性总结

### 去水印方法可行性矩阵

| 方法 | 图片 | 视频 | 画质 | 速度 | 难度 | 推荐 |
|------|------|------|------|------|------|------|
| 提取原始URL（doubao-nomark） | ✅ | ❌ | 无损 | 快 | 低 | ⭐⭐⭐ |
| 反向Alpha混合（lsj-watermark-remover） | ✅ | ❌ | 无损 | 快 | 中 | ⭐⭐⭐ |
| 豆包"变清晰"功能 | ✅ | ❌ | 无损 | 快 | 低 | ⭐⭐ |
| 开发者工具拦截水印请求 | ✅ | ❌ | 无损 | 快 | 中 | ⭐⭐ |
| 画面裁剪（FFmpeg） | ✅ | ✅ | 损失边缘 | 极快 | 低 | ⭐⭐ |
| AI修复（Morph Studio等） | ✅ | ✅ | 接近无损 | 慢 | 低 | ⭐⭐ |
| 官方会员导出 | ✅ | ✅ | 无损 | 快 | 低 | ⭐⭐⭐ |
| logo_type参数移除 | ❌ | ❌ | — | — | — | ❌ |
| lr参数操控 | ❌ | ❌ | — | — | — | ❌ |

### 最终建议

**对于视频去水印**：
1. **官方会员导出**是唯一能获取真正无损无水印视频的方法
2. **AI修复工具**（Morph Studio、HitPaw等）是当前最可行的后处理方案
3. **画面裁剪**适合水印在边缘的简单场景
4. **通过API获取无水印视频已被证实不可行**——所有标准API返回的视频都含水印

**对于图片去水印**：
1. **doubao-nomark** 或类似工具通过提取原始URL可完美去除水印
2. **lsj-watermark-remover** 的反向Alpha混合算法也是可靠的无损方案
3. 豆包APP内置的"变清晰"功能是官方隐藏技巧
