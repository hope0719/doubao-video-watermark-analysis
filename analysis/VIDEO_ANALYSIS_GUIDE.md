# 视频输出分析工具使用指南

## 🎯 目的
通过分析APK处理后视频的技术特征，推断其使用的去水印技术，即使静态分析未能发现相关代码。

## 📋 准备工作

### 1. 安装依赖
```bash
# macOS
brew install ffmpeg

# 检查安装
ffmpeg -version
```

### 2. 准备文件
- **原始视频文件**: 从豆包获取的带水印视频
- **处理后文件**: 使用APK处理后的去水印视频

## 🚀 使用方法

### 方法一：交互式使用
```bash
python3 tools/video_analysis_tool.py
```
按提示输入文件路径即可。

### 方法二：直接修改脚本
编辑脚本中的文件路径变量：
```python
original_file = "/path/to/original_video.mp4"
processed_file = "/path/to/processed_video.mp4"
```

## 📊 分析维度

### 1. 文件基本信息
- 文件大小对比
- 创建时间/修改时间
- 文件格式标识

### 2. 视频技术参数
- 分辨率（宽度 x 高度）
- 帧率（FPS）
- 编码格式（H.264等）
- 码率（Bitrate）
- 时长（Duration）

### 3. 文件结构特征
- 文件哈希值（MD5/SHA256）
- 文件头部结构
- 格式标识符

## 🔍 解读指南

### 如果检测到大变化：
- **解码再编码**: 视频被重新编码处理 → 可能是本地AI修复或云端处理
- **码率下降**: 画质压缩 → 可能是云端流媒体优化
- **分辨率变化**: 尺寸调整 → 可能是适配性优化

### 如果检测到小变化：
- **仅元数据修改**: 可能只修改了水印相关信息
- **相同编码格式**: 可能只是直接操作视频帧数据

## 💡 技术推论参考

### 推论 1：本地处理能力
- 视频在本地设备上被重新编码
- APK内置了视频处理算法
- 可能使用本地AI模型修复

### 推论 2：云端处理技术
- 视频上传到服务器处理
- 返回优化后的版本
- 可能结合多种AI技术

### 推论 3：协议级访问
- 直接获取了无水印源文件
- 绕过了正常的内容分发流程
- 可能发现了CDN特殊接口

## ⚠️ 注意事项

### 1. 技术限制
- 此分析只能通过输出结果推测技术，无法直接确认实现方式
- 需要结合其他分析方法进行验证

### 2. 准确性考虑
- 不同版本的水印可能处理效果不同
- 视频编码的复杂性可能导致分析误差
- 建议进行多次测试取平均值

### 3. 安全提醒
- 始终在法律框架内进行研究
- 保护个人隐私和数据安全
- 不要滥用技术分析结果

## 📈 实际应用示例

### 场景：发现码率下降但水印去除
```
原始文件: 1920x1080, H.264, 2159 kb/s
处理后文件: 1920x1080, H.264, 1200 kb/s
```
**推论**: 可能采用了云端流媒体优化技术，在去水印的同时降低了码率以节省带宽

### 场景：编码格式发生变化
```
原始文件: H.264, 1920x1080
处理后文件: H.265, 1920x1080  
```
**推论**: 可能重新编码以获得更好压缩率，同时进行水印修复

## 附录：常见问题

### Q: 如果没有FFmpeg怎么办？
A: 可以使用在线视频分析工具，或安装其他视频处理软件

### Q: 分析结果显示没有变化怎么办？
A: 可能水印去除效果不可见，或APK无效，建议重复测试

### Q: 如何保证分析的准确性？
A: 使用专业的视频分析工具，进行多次比对分析

### Q: 该工具支持哪些视频格式？
A: 支持MP4、MOV、AVI等主流格式，依赖FFmpeg的解码能力
