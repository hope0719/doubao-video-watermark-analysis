## 🎯 核心结论

**豆包视频的水印是在服务端 H.264 编码时像素级嵌入的，CDN 只存储一份带水印的视频文件。所有公开 API 返回的 URL 都指向同一个带水印版本。**

### ⚠️ 最终确认证据

经过3轮深度测试，我们获得了决定性证据：

1. **文件哈希完全一致**
   - MD5: `40b21e0f35b657a0e08a8f4f8d21cfdb`
   - SHA256: `daceb8d9b9e40718ef0f86d9af3d0d6042a99867eff14600d67e490042591a34`
   - 文件大小: `3,289,007 bytes`
   
2. **所有参数变体测试**
   - 测试了 14 种 lr 参数变体 → 全部返回相同文件
   - 测试了 11 种请求体字段组合 → 全部返回相同文件
   - 测试了 6 种 API 参数组合 → 全部返回相同文件
   - 测试了 4 种 CDN 路径变体 → 只有原始路径有效，其他403

3. **实际下载验证**
   - 下载了 4 个不同参数的完整视频文件
   - 计算MD5和SHA256完全一致
   - 证明确实是同一个文件的多个CDN副本

### 关键证据（已100%验证）

1. **文件哈希完全一致**（最强证据）
   - MD5: `40b21e0f35b657a0e08a8f4f8d21cfdb`
   - SHA256: `daceb8d9b9e40718ef0f86d9af3d0d6042a99867eff14600d67e490042591a34`
   - 所有参数变体下载的文件哈希100%相同

2. **ETag 一致性**
   - 标准ETag: `40b21e0f35b657a0e08a8f4f8d21cfdb`
   - 部分CDN节点返回N/A（不影响文件内容）
   - 实测证明：N/A只是响应头差异，文件内容完全相同

3. **文件大小固定**
   - 精确大小: `3,289,007 bytes`
   - 所有测试的文件大小完全一致

4. **lr 参数完全无效**
   - 测试了14种lr参数变体
   - 包括：`video_gen_no_watermark`, `origin`, `raw`, `clean`, `nowatermark` 等
   - 全部返回相同的MD5哈希

5. **请求体字段无效**
   - 测试了11种请求体组合
   - 包括：`watermark: false`, `remove_watermark: true`, `creator_download: true` 等
   - 全部返回相同的文件

6. **移动端/Web 端一致**
   - iOS、Android、Web、iPad 请求的哈希完全相同

## 🔬 技术分析

### 水印嵌入层级

```
┌─────────────────────────────────────┐
│  用户生成视频请求                      │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  服务端视频生成（AI 渲染）              │
│  └─ 原始视频帧                         │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  ★ 水印叠加层（像素级）                 │
│  └─ 在编码前混入视频帧                  │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  H.264 编码                           │
│  └─ 水印已永久嵌入编码流                │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  CDN 存储                             │
│  └─ 仅存储一份带水印的 MP4 文件         │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  客户端播放/下载                       │
│  └─ 只能获取带水印版本                  │
└─────────────────────────────────────┘
```

### 关键技术点

1. **编码级嵌入**：水印不是通过 CSS overlay 或动态叠加，而是在编码前混入像素
2. **单一文件存储**：CDN 上只有一个版本，所有 URL 参数只是缓存 key
3. **参数欺骗性**：`lr=video_gen_watermark_dyn` 看起来像是控制水印，实际只是标记
4. **API 一致性**：所有公开 API 都指向同一个 CDN 文件

**问题**：
