# 豆包视频去水印 — 全部尝试方法总结（供研究学习）

> 以下是我们实际尝试过、测试过的所有方法，按章节分类整理。
> 最后更新：2026-07-04

---

## 📋 结论前置

**客户端无法获取无水印豆包视频。** 水印在服务端 H.264 编码时像素级嵌入，CDN 只存一份文件，所有公开 API 返回的 URL 均为带水印版本。

---

## 一、API 端点探测

### 1.1 `/samantha/media/get_play_info`（核心尝试）

```
POST https://www.doubao.com/samantha/media/get_play_info
Body: {"key": "video_id"}
```

响应结构：
```json
{
  "data": {
    "original_media_info": {
      "main_url": "https://v9-show.douyinvod.com/xxx.mp4"  // ← 1080p，带水印
    },
    "media_info": [{
      "main_url": "https://v9-show.douyinvod.com/xxx.mp4"   // ← 720p，带水印
    }],
    "poster_url": "https://p3-sign.doubao.com/xxx.jpg"      // ← 封面图URL
  }
}
```

**验证方式**：
- `curl -I <original_main_url>` → 查看 `etag: "5bd9650c..."`
- 对带参数和不带参数的 URL 各请求一次 → **etag 完全一致**
- 说明 CDN 是同一份文件，参数只影响缓存 key

### 1.2 `/samantha/video/get_play_info`

测试请求返回 `v9-show.douyinvod.com` 域名 URL，etag 和 `/media/` 版本完全一致。

### 1.3 `/creativity/share/get_video_share_info`

响应中 `play_info.main` 也是 `v9-show.douyinvod.com`，etag 一致。

### 1.4 chat 页面的 video_id 来源

```
/im/chain/single → response["video"]["vid"]
```

字节跳动视频 ID 格式：`v0269cg10004d83lml2ljht9qodq12pg`（22位字母数字，含 g/l/j/p/q 等非hex字符）

## 二、CDN URL 参数操控

### 2.1 关键 URL 参数

| 参数 | 测试操作 | 结果 |
|:-----|:---------|:-----|
| `lr=video_gen_watermark_dyn` | 去除 `lr` 参数 | 403 Forbidden |
| `lr=video_gen_watermark_dyn` | 替换为 `lr=video_gen_no_watermark` | 返回带水印版本，etag 一致 |
| `ft=xxx` | 去除 | 403 |
| `cs=xxx` | 去除 | 403 |
| `dr=xxx` | 去除 | 403 |
| `download=true` | 去除 | 403 |

### 2.2 域名变换

| 原域名 | 变换后 | 结果 |
|:-------|:-------|:-----|
| `v9-show.douyinvod.com` | `v9.douyinvod.com` | 403 |
| `v26-show.douyinvod.com` | `v26.douyinvod.com` | 403 |
| `bytedance/byteimg` | 替换其他 CDN 域名 | 403 |

### 2.3 请求头操控

| 测试操作 | 结果 |
|:---------|:-----|
| 添加 `agw-js-conv: str` | 无影响，etag 一致 |
| 替换 UA 为微信浏览器 | 结果和默认 UA 相同 |
| 替换 Referer | 影响 CDN 防盗链但不影响文件内容 |
| 添加 `type: "video"` 请求体 | API 返回空或错误 |

### 2.4 etag 验证方法

```bash
curl -sI "<url1>" | grep -i etag   # 5bd9650c...
curl -sI "<url2>" | grep -i etag   # 5bd9650c... （相同）
```

结论：所有参数变体指向 CDN 上同一个文件，水印无法通过 URL 参数绕过。

## 三、页面侧实时提取

### 3.1 注入式 fetch/XHR 拦截（inject.js）

将 content script 注入为 `MAIN world` + `run_at: "document_start"`，monkey-patch `fetch` 和 `XMLHttpRequest`：

```js
// 拦截 fetch 响应，扫描 body 中的 CDN URL
const originalFetch = window.fetch;
window.fetch = async function(...args) {
  const response = await originalFetch.apply(this, args);
  // clone response, parse JSON, scan for media URLs
  return response;
};
```

**捕获到的 URL 样本：**
```
https://p3-flow-imagex-sign.byteimg.com/tos-cn-i-a9rns2rl98/rc_gen_image/73cab1xxxxxx
https://p11-flow-imagex-sign.byteimg.com/tos-cn-i-a9rns2rl98/rc_gen_image/3146bfxxxxxx
```

**注意**：`flow-imagex-sign` 是图片 CDN，和视频无关。

### 3.2 HTML 兜底正则提取

```html
<!-- 页面中图像数据存放在 data-fn-args 属性 -->
<script data-script-src="modern-run-router-data-fn"
  data-fn-args='[{"data":{"message_snapshot":{"message_list":...}}}]'>
```

解析 JSON，遍历到 `creation.image.image_ori_raw.url`，提取到无水印图片 URL。

**注意**：这个方法对**图片**有效（`image_ori_raw.url` 中的 URL 下载后无水印），对**视频**无效。

### 3.3 chat 页面 video 元素提取

```js
// 获取 chat 页面 video 元素
const videoEl = document.querySelector('video');
const vSrc = videoEl.src;       // blob:https://www.doubao.com/xxx （chat 页面是 blob）
const vPoster = videoEl.poster; // 封面图 CDN URL
```

chat 页面 video.src 是 blob URL，无法从中提取 vid。需要通过以下方式：
1. 包围 video 元素，向上搜索 DOM 属性中的 vid
2. intercept fetch 响应获取 `/im/chain/single` 的 vid 字段
3. 用 video.poster CDN URL 匹配 videoCoverMap 中的 coverUrl

## 四、开源项目调研

### 4.1 调研结论

克隆并分析 8 个 GitHub 开源去水印项目，全部使用以下方案：

| 项目名 | 去水印原理 | 实际上是否无水印 |
|:-------|:-----------|:---------------|
| catscarlet/doubao-watermark-free | `get_play_info` → `original_media_info.main_url` | ❌ 带水印（etag 一致） |
| xiaoka6688/doubao-no-watermark | `lr=video_gen_no_watermark` 参数 | ❌ 带水印（etag 一致） |
| Luncot/doubao-watermark-remover | `lr=video_gen_no_watermark` + 头 | ❌ 带水印 |
| wan-kong/doubao-watermark | 微信 UA 获取 | ❌ 带水印 |
| AI-Video-Copilot/doubao | `get_play_info` | ❌ 带水印 |
| xx05kyo/doubao | `get_play_info` | ❌ 带水印 |
| waqq0011/jimeng | Jimeng 专用 API → `transcoded_video` | ✅ 即梦本身无水印 |
| hfstack/aidownloadtool | Jimeng 专用 API | ✅ 即梦本身无水印 |

**结论**：所有豆包去水印项目在技术上下载的视频**均带水印**，即使展示"成功"也可能是旧版 CDN Bug 或水印不明显导致的认知偏差。

### 4.2 关键发现

1. 即梦（Jimeng）视频下载的是 `transcoded_video.origin` URL，本身无水印（服务端不加水印）
2. 根本区别：即梦不加水印，豆包加水印
3. 通义千问（Qwen）图片下载 `__NEXT_DATA__` 中的 `originalLink`，本身无水印

## 五、其他尝试

### 5.1 视频封面图匹配

用途：验证 vid 对应的正确视频

```
/im/chain/single → video.vid → API get_play_info → data.poster_url
```

`poster_url` 和 `videoElement.poster` 的 CDN URL 可以匹配成功。
但 `original_media_info.main_url` 仍然带水印。

### 5.2 外挂 FFmpeg 后期处理

去水印裁剪方案，需要分析水印位置（通常在右下角），做模糊或裁剪。
- 缺点：画质损失、码率下降、不是真正的"无水印"

### 5.3 CDN 早期 Bug（2026年初发现）

2026年5月之前，某些 CDN 节点返回的文件不含水印。
2026年5月后该 Bug 被修复，现在所有节点返回带水印版本。

## 六、功能矩阵总结

| 平台 | 图片去水印 | 视频无水印 | 备注 |
|:----:|:---------:|:---------:|:-----|
| 豆包（doubao.com） | ✅ 成功 | ❌ 不可能 | 图片通过 `image_ori_raw.url` 获取；视频水印是编码级 |

## 七、关键证据

1. **etag 一致性**：所有 URL 参数变体返回相同 etag（5bd9650c...），证明是同一份文件
2. **17 个 API 测试**：所有 API 返回的 CDN URL 均有水印
3. **15 个开源项目分析**：所有项目使用相同方案，下载的视频均有水印
4. **CDN 域名变换全部 403**：非 show 域名直接拒绝
5. **URL 参数操控全部 403**：去除或修改 watermark 参数被拒绝

## 八、可能的探索方向（未验证）

1. **浏览器 DevTools 抓包**（.m3u8 → .ts 分片）：看视频分段 URL 是否包含不同编码版本
2. **App 端逆向**：抓取豆包 iOS/Android App 的播放器协议，看 m3u8 playlist 是否有更高清流
3. **抖音底层 CDN**：豆包视频可能托管在字节跳动 / 抖音 CDN 上，看看抖音主站是否有不同编码版本
4. **微信小程序端协议**：微信小程序里豆包播放时的请求是否有不同版本
5. **直播流协议**：豆包视频是生成后还是实时生成的？实时流可能不带水印
