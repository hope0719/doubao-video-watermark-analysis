## 方法1：浏览器自动监听（最简单）⭐⭐⭐⭐⭐

### 适合：任何人，无需技术背景

### 步骤：

**1. 打开豆包网站**
```
在Chrome中访问：https://www.doubao.com
```

**2. 打开开发者工具**
```
按 F12（Windows）
或 Cmd+Option+I（Mac）
```

**3. 进入Console标签**
```
点击顶部的 "Console" 标签
```

**4. 复制监听脚本**
```
打开文件：inject_monitor.js
全选复制（Ctrl+A, Ctrl+C 或 Cmd+A, Cmd+C）
```

**5. 粘贴到Console并运行**
```
在Console中粘贴（Ctrl+V 或 Cmd+V）
按 Enter 键
看到 "监听已注入" 提示
```

**6. 操作网页**
```
在豆包网站上播放视频
或刷新页面
或进行任何操作
```

**7. 查看输出**
```
Console中会自动显示：
- 所有API调用
- WebSocket连接
- 包含 unwatermarked 的请求
```

**8. 截图或复制**
```
找到包含 unwatermarked 的输出
右键 -> 复制
或截图
发给我分析
```

### 预期结果：
```
🔍 发现 Fetch 请求:
URL: https://api.xxx.com/parse
Method: POST
Headers: {...}
Body: {...}
Response: {"video_url": "...unwatermarked..."}
```

## 方法2：Stream抓包完整版（最准确）⭐⭐⭐⭐⭐

### 适合：已经会用Stream的用户

### 关键：找到视频URL **之前** 的API调用

### 步骤：

**1. 完全清空Stream**
```
打开Stream App
点击右上角的"清空"按钮
确认已清空所有记录
```

**2. 打开小程序**
```
打开微信
进入去水印小程序
等待完全加载
```

**3. 开始操作**
```
粘贴豆包视频链接：
https://www.doubao.com/video-sharing?share_id=xxx

点击"解析"或"去水印"按钮
```

**4. 等待完成**
```
等待小程序显示视频或下载按钮
不要点击下载！
```

**5. 回到Stream**
```
按时间从早到晚查看请求
关注 doubao.com 域名的请求
```

**6. 识别API调用的特征**

找到符合以下特征的请求：

| 特征 | 说明 |
|:-----|:-----|
| 时间 | 最早的几个请求 |
| 域名 | *.doubao.com 或其他API域名 |
| 路径 | 包含 parse/get_url/media 等 |
| 类型 | JSON 响应（不是video/mp4） |
| 大小 | 很小（几KB），不是几百KB |

**7. 检查响应内容**

点击可疑的请求：
- 查看响应体（Response Body）
- 搜索 "unwatermarked"
- 搜索 ".mp4"
- 搜索 "video_url"

**8. 记录完整信息**

找到后，点击请求，复制：
```
1. 请求URL（完整）
2. 请求方法（GET/POST）
3. 所有Headers
4. Request Body（如果有）
5. Response Body（完整）
```

### 时间线参考：

```
时间轴示例：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
18:06:13  API调用（这个是我们要找的！）
          ↓ 返回 unwatermarked URL
18:06:14  下载视频文件（你已提供的）
          ↓ 文件内容
18:06:15  下载完成
```
