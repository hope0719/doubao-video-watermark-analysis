# 豆包视频去水印 — 完整测试结论（2026-07-06 更新）

> 基于真实登录 Cookie 的实际探测结果

---

## 一、核心结论（已确认）

### ✅ 结论 1：`original_media_info` 已不再提供无水印 URL

**测试方式**：用真实登录 Cookie 调用 `POST /samantha/media/get_play_info`

**结果**：
```json
{
  "media_info": [{"main_url": "https://v9-videoweb.doubao.com/...&lr=video_gen_watermark_dyn&..."}],
  "original_media_info": {"main_url": "https://v9-videoweb.doubao.com/...&lr=video_gen_watermark_dyn&..."}
}
```

`original_media_info.main_url` 与 `media_info[0].main_url` **完全相同**，均含 `lr=video_gen_watermark_dyn`。

**含义**：豆包服务端已改变逻辑，不再为 `original_media_info` 提供单独的无水印地址。这与早期报告（2024 年）的描述不同，说明豆包已修复了这个"漏洞"。

---

### ✅ 结论 2：修改 CDN 视频 URL 中的 `lr` 参数无效

**测试方式**：对同一个视频文件，请求 5 种不同 `lr` 参数：
1. `lr=video_gen_watermark_dyn`（原始）
2. `lr=no_watermark`
3. `lr=origin`
4. `lr=unwatermarked`
5. 去掉 `lr` 参数

**结果**：所有请求返回 **完全相同的文件**（同样是 1097361 字节），`Content-Range` 完全一致。

**含义**：`lr` 参数是服务端生成 URL 时的一个标记，但 CDN 存储的只有一份文件（带水印版本）。修改 `lr` 不会让 CDN 返回不同文件。

---

### ✅ 结论 3：`video/get_play_info` 端点需要 HttpOnly Cookie

**现象**：用户浏览器（已登录）能成功调用 `POST /samantha/video/get_play_info` 并返回 200，但用 `document.cookie` 导出的 Cookie 调用同一端点，返回 `code: 710012001（登录已过期）`。

**原因**：`document.cookie` 无法读取 `HttpOnly` 标记的 Cookie。豆包的鉴权机制中，`video/get_play_info` 需要某个 HttpOnly Cookie（可能是 `session_id` 或类似字段），而这个 Cookie 无法通过 JavaScript 读取。

**影响**：我们无法用 Python 脚本完全模拟浏览器的 `video/get_play_info` 请求，除非能用浏览器扩展导出完整 Cookie（含 HttpOnly）。

**但注意**：即使用户浏览器成功调用了这个端点，返回的 `play_infos[0].main` 同样含 `lr=video_gen_watermark_dyn`（见用户的 Monitor JSON）。所以即使能调通，也不保证能拿到无水印 URL。

---

### ✅ 结论 4：所有公开 API 端点均已穷尽

已测试的登录态端点（全部返回 200，但无水印 URL）：

| 端点 | 状态码 | 有无视频 URL | 有无无水印字段 |
|------|--------|-------------|---------------|
| `/samantha/media/get_play_info` | 200 | ✅ 有（带 `lr=watermark`） | ❌ `original_media_info` 与 `media_info` 相同 |
| `/creativity/share/get_video_share_info` | 200 | ✅ 有（带水印） | ❌ |
| `/samantha/video/get_play_info` | 710012001 | ❌ 登录过期 | — |
| `/samantha/video/query_video_gen_info` | 200 | ❌ 无视频 URL | — |
| `/samantha/media/mget_play_status` | 200 | ❌ 无视频 URL | — |
| `/alice/resource/watermark_task` | 200 | ❌ 返回空 body | — |
| `/alice/resource/watermark_download` | 200 | ❌ 返回空 body | — |

**关键发现**：`watermark_task` 和 `watermark_download` 端点虽然存在且返回 200，但响应体为空或不含有效数据。这说明这些端点可能是"预留"接口，当前版本未启用，或者需要更高权限才能调用。

## 二、剩余可行方向

### 方向 A：获取完整 Cookie（含 HttpOnly）后重新测试 `video/get_play_info`

**原理**：`video/get_play_info` 是用户浏览器实际使用的端点，且返回格式（`play_infos`）与 `media/get_play_info`（`media_info`）不同。可能 `play_infos` 里有不同的字段。

**操作步骤**：
1. 在 Chrome/Edge 中安装 Cookie-Editor 扩展
2. 打开 `www.doubao.com` 并登录
3. 点击 Cookie-Editor 扩展图标 → 导出为 JSON（Netscape 格式或 JSON 格式）
4. 把完整 Cookie 文件发给我
5. 用完整 Cookie 调用 `video/get_play_info`，分析 `play_infos` 完整响应

**成功率预估**：⭐⭐（即使调通，Monitor JSON 显示返回的 URL 同样带 `lr=watermark`）

---

### 方向 B：让豆包生成一个新的视频，测试"我的创作"页面

**原理**：之前测试的视频 ID 可能来自"分享链接"（他人生成的视频）。豆包可能对"自己生成的视频"提供不同的下载权限（例如创作者的"下载"按钮可能有无水印版本）。

**操作步骤**：
1. 在豆包网页端生成一个新视频
2. 在"我的创作"页面找到这个视频
3. 点击"下载"按钮，同时用 F12 Network 抓包
4. 看下载请求返回的 URL 是否含 `lr=watermark`

**成功率预估**：⭐⭐⭐（最有可能存在差异的地方）

---

### 方向 C：AI 去水印（技术可行，需工具）

**原理**：水印是像素级烘焙的，可以用 AI inpainting 技术逐帧修复。

**工具选项**：
1. **ComfyUI + ProPainter 节点**（自托管，免费，需要 GPU）
2. **Morph Studio**（在线，有免费额度）
3. **HitPaw Video Object Remover**（付费，效果较好）

**成功率预估**：⭐⭐⭐⭐（技术可行，效果取决于水印复杂度）

---

### 方向 D：画面裁剪（最简单，立即可用）

**原理**：豆包水印一般固定在右下角，用 FFmpeg 裁剪掉边缘即可。

**操作**：
```bash
# 假设水印在右侧 80 像素宽度区域
ffmpeg -i input.mp4 -vf "crop=iw-80:ih:0:0" -c:a copy output.mp4
```

**需要先做的**：下载一个带水印的视频样本，用 FFmpeg 提取第一帧，测量水印的精确位置（像素坐标）。

**成功率预估**：⭐⭐⭐⭐⭐（最简单，立即可以验证）

---

### 方向 E：联系豆包官方 / 查看创作者权益

**原理**：豆包可能向付费用户或创作者提供无水印导出功能（类似剪映的导出）。

**操作步骤**：
1. 在豆包网页端查看"会员"/"创作者中心"选项
2. 查看是否有"高清导出"/"无水印导出"功能
3. 联系豆包客服询问

**成功率预估**：⭐⭐⭐⭐（合法合规，如果官方提供支持的话）
