# 即梦（Dreamina / Jimeng）去水印方法

> 验证日期：2026-05-31
> 适用：即梦 Web 作品分享链接（短链 `https://jimeng.jianying.com/s/...` / `https://m.jimeng.jianying.com/s/...`）
> 状态：✅ 已发布作品可用；❌ 邀请/裂变链接无作品可取

---

## 0. 一句话结论

即梦的分享短链有**两类**，靠跳转后的 `share_campaign_key` 区分：

| 类型 | `share_campaign_key` | 能否取到作品 | 说明 |
|---|---|---|---|
| **已发布作品分享** | `jimeng_share_v2` | ✅ 能 | 走 `get_item_info` 拿无水印视频/原图 |
| **邀请/模板裂变** | `dreamina_invite_fission` | ❌ 不能 | "注册送积分/邀请码"活动页，不含作品 |

只有 `jimeng_share_v2` 这类指向**已发布作品**的链接才能去水印。`dreamina_invite_fission`
（`scene=try_template`、`item_type=local`）是拉新裂变链接，对应的是别人本地的草稿/模板，
即梦作品库里查不到（`itemId not exist`），没有可下载的视频/图片文件。

---

## 1. 背景

即梦分享出来的链接通常是一条短链，文案形如：

```
看我在即梦发现了什么！XXX发布了一篇AI作品，快来看吧！😆 https://jimeng.jianying.com/s/Ctnjnn5fmh8/?t=9 BA6390 ...
```

或

```
看我创作的AI作品！加入即梦，注册即得免费积分，开启AI创作！😆 https://m.jimeng.jianying.com/s/CUgNUXlmUGQ/?t=53 AA7543 ...
```

光看文案不可靠（两种都说"AI作品"），**必须 follow 短链看跳转参数**才能判断类型。

### 旧做法已失效

项目里 `utils/api.js` 现有的 `jimengVideoAnalysis` 走的是
`POST .../luckycat/cn/jianying/campaign/v1/pippit/share/landing_page` 接口，
传 `share_id` / `share_sec_did` / `share_sec_uid`。对现在的新版分享链接，
**这个接口对两类链接统一只返回兜底页**（`fallback_page`，引导"去打开小云雀/下载App"），
里面没有 `video_info` / `image_info`，拿不到任何作品。

---

## 2. 第一步：解析短链

短链会 302 跳转到带完整参数的落地页。用任意 UA follow 一次拿 `Location` 头即可（不需要登录）。

```bash
curl -sI -A "Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) ..." \
  "https://jimeng.jianying.com/s/Ctnjnn5fmh8/?t=9" | grep -i '^location'
```

### 跳转后两类链接的参数对比

**已发布作品（可取）：**
```
https://jimeng.jianying.com/activities/reflux/mproject?...
  &id=7633266505479851289            ← 作品 ID（关键）
  &scene=cast_template               (或 scene=template)
  &share_campaign_key=jimeng_share_v2   ← 判定依据
  &share_token=993a7cb6-...          (可选，接口其实不需要)
```

**邀请/裂变（不可取）：**
```
https://m.jimeng.jianying.com/activities/share?...
  &id=7621068196719660298
  &item_id=7621068196719660298
  &item_type=local                   ← local = 本地草稿/模板
  &publish_type=local
  &scene=try_template
  &share_campaign_key=dreamina_invite_fission  ← 判定依据
```

### 提取关键参数

```js
// 从跳转后的 URL 里取
var itemId      = url.match(/[?&]id=(\d+)/)[1];
var campaignKey = (url.match(/share_campaign_key=([^&]+)/) || ['',''])[1];
// campaignKey === 'jimeng_share_v2'        -> 走第 3 节解析
// campaignKey === 'dreamina_invite_fission' -> 邀请链接，直接提示无作品
```

## 3. 真正的作品接口

通过反编译活动页的 webpack chunk（`reflux.70eef7f7.js` 引用的
`3455.b697ee42.js`），在 `JianyingMWebServerService` 里找到作品详情接口：

```
POST https://jimeng.jianying.com/mweb/v1/get_item_info?aid=513695&device_platform=web&region=CN&web_id=<任意web_id>
```

### 关键点

| 项 | 值 |
|---|---|
| 必需 query | `aid=513695`（即梦 Web 的 app id） |
| 必需 body | `{"published_item_id": "<id>"}`（**字段名是 `published_item_id`，不是 `item_id` / `vid`**） |
| 推荐 body | 额外带 `pack_item_opt:{}` 和 `item_not_find_detail:true` |
| 是否需要登录 | ❌ 不需要 |
| 是否需要 `share_token` | ❌ 不需要 |
| Content-Type | `application/json` |

> 踩坑记录：用 `item_id` / `item_ids` / GET 方式都会报 `ret 1000: invalid parameter`，
> 只有 `published_item_id`（从 chunk 里 `GetItemInfo` 的 data 结构挖到）才对。
> 批量版接口是 `POST /mweb/v1/mget_item_info`，body 用 `item_id_list: [...]`。

### 请求示例（curl）

```bash
curl -s "https://jimeng.jianying.com/mweb/v1/get_item_info?aid=513695&device_platform=web&region=CN&web_id=7645896869637473846" \
  -X POST \
  -H "content-type: application/json" \
  -H "origin: https://jimeng.jianying.com" \
  -H "referer: https://jimeng.jianying.com/" \
  -A "Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 Mobile/15E148 Safari/604.1" \
  -d '{"published_item_id":"7633266505479851289","pack_item_opt":{},"item_not_find_detail":true}'
```

### 返回码

| `ret` | 含义 |
|---|---|
| `"0"` | 成功，`data` 里有作品 |
| `"2032"` | `itemId not exist`——id 不是已发布作品（邀请/裂变链接就是这种） |
| `"1000"` | `invalid parameter`——参数名/方式不对 |

## 4. 响应结构

`data` 顶层关键字段：

```
data
├── common_attr      // id / title / cover_url / cover_url_map ...
├── author           // 作者信息
├── video            // 视频作品时存在
├── image            // 图片作品时存在
├── aigc_image_params// AIGC 生成参数（prompt 等）
└── ...
```

通过 `data.video` / `data.image` 哪个存在判断作品类型。

### 4.1 视频作品 `data.video`

```
data.video
├── video_id        : "v02870g10004d7nq..."
├── duration        : 13
├── origin_video    : { video_url, width, height, size, md5, ... }   // 原始上传源
└── transcoded_video
    ├── origin : { video_url, width:720, height:1254, size, ... }     // 最高画质（推荐）
    ├── 720p   : { video_url, ... }
    ├── 480p   : { video_url, ... }
    └── 360p   : { video_url, ... }
```

**取 URL 的优先级（带回退）：**

```js
var tv = data.video.transcoded_video || {};
var downurl =
    (tv.origin   && tv.origin.video_url)   ||
    (tv['720p']  && tv['720p'].video_url)  ||
    (tv['480p']  && tv['480p'].video_url)  ||
    (tv['360p']  && tv['360p'].video_url)  ||
    (data.video.origin_video && data.video.origin_video.video_url) || '';
```

实测：`origin` 档 720×1254、H.264、约 6.4 Mbps、13s → 下载得到合法无水印 mp4。

### 4.2 图片作品 `data.image`

```
data.image
├── format       : "png"
└── large_images : [
      {
        image_uri : "tos-cn-i-tb4s082cfz/84ecddb8...",
        image_url : "https://p26-dreamina-sign.byteimg.com/...aigc_resize:0:0.png?...",
        width : 1440, height : 2560, size, format
      },
      ...   // 图集会有多张
    ]
```

**取原图：**

```js
var images = (data.image.large_images || []).map(function (it) { return it.image_url; });
```

实测：`large_images[0]` → 1440×2560 PNG，下载得到无水印原图。

## 5. 完整解析流程

```
短链 /s/xxx
   │  follow 302
   ▼
落地页 URL ── 解析 id + share_campaign_key
   │
   ├── campaign_key == 'dreamina_invite_fission'  →  邀请/裂变链接，提示"无可下载作品"，结束
   │
   └── campaign_key == 'jimeng_share_v2'
          │  POST /mweb/v1/get_item_info  {published_item_id: id}
          ▼
       ret==0 ? ──否(2032/其他)── 提示失败/作品不存在
          │是
          ▼
       data.video 存在 → 取 transcoded_video.origin (带清晰度回退)  → 视频无水印 URL
       data.image 存在 → 取 image.large_images[].image_url           → 图片无水印 URL
```

### 参考伪代码（可落地到 `utils/api.js`）

```js
jimengAnalysisV2: function (shareUrl, onSuccess, onFail) {
  // 1. follow 短链拿落地页 URL（小程序里用 wx.request 拿 res.header.Location）
  resolveRedirect(shareUrl, function (landingUrl) {
    var idM  = landingUrl.match(/[?&]id=(\d+)/);
    var keyM = landingUrl.match(/share_campaign_key=([^&]+)/);
    var id   = idM ? idM[1] : '';
    var key  = keyM ? keyM[1] : '';

    if (!id) return onFail('无法识别作品ID');
    if (key === 'dreamina_invite_fission') {
      return onFail('该链接是邀请/活动页，不含可下载作品');
    }

    // 2. 调作品接口
    wx.request({
      url: 'https://jimeng.jianying.com/mweb/v1/get_item_info?aid=513695&device_platform=web&region=CN&web_id=7645896869637473846',
      method: 'POST',
      header: {
        'content-type': 'application/json',
        'origin': 'https://jimeng.jianying.com',
        'referer': 'https://jimeng.jianying.com/'
      },
      data: { published_item_id: id, pack_item_opt: {}, item_not_find_detail: true },
      success: function (res) {
        var j = res.data || {};
        if (j.ret !== '0' || !j.data) return onFail(j.errmsg || '作品不存在');
        var d = j.data;

        if (d.video) {
          var tv = d.video.transcoded_video || {};
          var url = (tv.origin && tv.origin.video_url) ||
                    (tv['720p'] && tv['720p'].video_url) ||
                    (tv['480p'] && tv['480p'].video_url) ||
                    (tv['360p'] && tv['360p'].video_url) ||
                    (d.video.origin_video && d.video.origin_video.video_url) || '';
          if (!url) return onFail('未获取到视频地址');
          return onSuccess({ type: 'video', downurl: url,
                             cover_url: (d.common_attr && d.common_attr.cover_url) || '',
                             duration: d.video.duration || 0 });
        }

        if (d.image) {
          var imgs = (d.image.large_images || []).map(function (x) { return x.image_url; });
          if (!imgs.length) return onFail('未获取到图片');
          return onSuccess({ type: 'images', images: imgs, cover_url: imgs[0] });
        }

        onFail('未识别到视频或图片');
      },
      fail: function () { onFail('网络错误'); }
    });
  });
}
```

## 6. 注意事项

### URL 时效性
- 视频 URL（`v26-artist.vlabvod.com/...`）和图片 URL（`*-dreamina-sign.byteimg.com/...`）
  都是**带签名的临时链接**，`dy_q` 是时间戳、`x-expires` / `x-signature` 控制有效期，**约 1 小时过期**。
- **不要持久化保存 URL**，每次需要播放/下载时重新调 `get_item_info` 取新签名。

### 反爬 / 风控
- `get_item_info` 必须带 `?aid=513695`。
- UA / Origin / Referer 不是强制，但建议带上 `referer: https://jimeng.jianying.com/`，避免被新风控拦。
- 不要短时间高频打同一个 id，可能触发限流。

### 字段名易混
| 概念 | 正确字段 | 错误写法 |
|---|---|---|
| 作品 ID（请求体） | `published_item_id` | ~~`item_id`~~ / ~~`vid`~~ |
| 批量请求体 | `item_id_list: []`（用 `mget_item_info`） | — |
| 视频地址 | `data.video.transcoded_video.origin.video_url` | — |
| 图片地址 | `data.image.large_images[].image_url` | — |

### `landing_page` 老接口
现有 `jimengVideoAnalysis` 的 `landing_page` 路径对新版分享链接已失效（只回 fallback）。
建议保留作旧链接兜底，新链接统一走第 5 节的 `get_item_info` 流程。

---

## 7. 失败兜底（接口变动时的排查方向）

1. **重跑反编译流程**：
   - 拉落地页 `https://jimeng.jianying.com/activities/reflux/mproject?...` 的 HTML
   - 在 HTML 里找 `lf3-lv-buz.vlabstatic.com/.../activities/static/js/` 下的 chunk 列表
   - 下载各 chunk，`grep get_item_info` 定位，读 `JianyingMWebServerService` 里对应方法的 `data` 结构，确认最新请求字段
2. 用最小请求集 + 二分法定位**强制 query**（如 `aid`）。
3. 抓 App 端流量（mitmproxy/Reqable），App 接口往往比 Web 多一套且参数不同，Web 失效时可改 UA 模拟 App。
