## 三、综合结论

1. **旧版参数回退无效**：`version_code` 和 `aid` 的历史值都无法恢复无水印字段。当前所有公开未登录接口返回的都是带水印版本。

2. **web JS 中不存在独立的任务创建端点**：`watermark_task` 和 `watermark_download` 依赖外部传入 `task_id`，而当前 bundle 没有定义如何创建任务。

3. **登录态端点是目前唯一未验证的盲区**：
   - `/samantha/video/get_play_info`
   - `/samantha/video/query_video_gen_info`
   - `/alice/resource/watermark_task`
   - `/alice/resource/watermark_download`

   这些端点在未登录时都返回“登录已过期”，说明它们需要有效的 Cookie/Token。如果存在获取无水印视频的官方内部路径，极有可能在这几个端点之中。

4. **下一步建议**：
   - 在**已登录浏览器环境**中（通过 Edge 插件或油猴脚本）调用这些端点，查看响应是否包含无水印 URL 或不同的 `lr` 参数。
   - 在视频生成过程中，监听这些端点是否被调用，以及调用参数是什么。
   - 对桌面端/APP 进行抓包，确认是否使用 `/samantha/video/*` 而非 `/samantha/media/*`。
