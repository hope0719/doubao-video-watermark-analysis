二、分析方法与工具
────────────────────────────────────────────────────────────────
1. agent-browser（无头浏览器，Chromium 内核）
   - 模拟移动端访问分享页，等待 SPA 渲染完成
   - 通过 snapshot 获取页面 DOM 结构
   - 通过 eval 执行 JS 提取视频元素 src

2. CDP（Chrome DevTools Protocol）
   - 通过 WebSocket 连接浏览器调试端口
   - 使用 Network.enable 监听所有网络请求/响应
   - 使用 Network.getResponseBody 获取 API 响应体
   - 使用 Network.requestWillBeSent 捕获请求头和 POST body
   - 使用 suppress_origin 绕过 WebSocket 403 限制

3. JS 源码逆向分析
   - 下载页面加载的全部 JS bundle（16 个文件，共约 1.5MB）
   - 重点分析 42472.b1ed7bae.js（393KB，API 定义文件）
   - 提取所有 API 端点定义、参数结构、调用方式

4. VLM 视觉语言模型
   - 使用 glm-4.6v 识别视频帧中的水印位置和内容
   - 对比不同 API 返回的视频帧水印差异


三、页面架构分析
────────────────────────────────────────────────────────────────
分享页为纯 SPA（单页应用），服务端返回的 HTML 仅包含骨架屏和 JS 引用，
视频信息通过前端 JS 动态调用 API 获取。

页面加载流程：
  1. 加载 HTML 框架 → 2. 加载 JS bundle → 3. 调用 get_video_share_info API
  → 4. 获取视频 URL → 5. 渲染 <video> 元素 → 6. 开始播放

安全机制：
  - msToken：动态生成的请求签名令牌
  - ttwid：设备追踪 Cookie
  - bdms.js / sdk-glue.js：请求签名 SDK
  - webmssdk.es5.js：WebSocket 安全通信 SDK
  - Agw-Js-Conv: str 请求头：API 网关约定头


四、发现的 API 端点
────────────────────────────────────────────────────────────────
通过 JS 源码逆向，在 42472.b1ed7bae.js 中发现以下相关 API：

  【核心视频 API】
  1. POST /creativity/share/get_video_share_info
     参数：{share_id, vid, download_params, creation_id}
     用途：分享页视频信息（页面默认调用的 API）

  2. POST /samantha/video/get_play_info
     参数：{vid}
     用途：内部播放信息（豆包 App 内部使用的播放接口）

  3. POST /samantha/video/query_video_gen_info
     参数：{vid, share_id}
     用途：查询视频生成状态

  4. POST /samantha/media/get_play_info
     参数：{key, share_id, node_id}
     用途：媒体播放信息（需要 node_id，无法外部调用）

  【下载相关 API】
  5. POST /alice/resource/watermark_download
     参数：{uri, url, scene, sub_scene, content_type, file_type}
     注意：参数作为 query params 传递，非 request body
     用途：水印资源下载（返回 download_url 和 task_id）

  6. GET /alice/resource/watermark_task
     参数：{task_id}
     用途：水印任务轮询

  7. POST /samantha/aispace/get_download_info
     参数：{requests: [{uri, content_type}]}
     用途：AI 空间下载信息

  8. POST /samantha/code/get_resource_download_url
     参数：{resource_id, resource_type(int)}
     用途：代码资源下载

  9. POST /samantha/creation/artifact/get
     参数：{id(int)}
     用途：创作工件获取（需登录）

  10. POST /alice/media/bigmusic/get_video
      参数：{video_id, share_id, download_params, template_id, creation_id}
      用途：大音乐视频信息

  【其他相关 API】
  11. POST /alice/resource/get_config — 资源配置
  12. POST /alice/resource/register — 资源注册
  13. POST /samantha/creation/artifact/get_meta_by_resource — 工件元信息
  14. POST /alice/message/douyin_playback — 抖音回放

八、download_params 参数测试
────────────────────────────────────────────────────────────────
get_video_share_info API 支持 download_params 字段：

  传入值               结果
  ──────────────────────────────────────────
  不传（undefined）     ✅ 成功，返回带水印 URL
  {}（空对象）          ✅ 成功，返回带水印 URL
  ""（空字符串）        ❌ system error
  true                  ❌ system error
  1                     ❌ system error
  "no_watermark"        ❌ system error
  {no_watermark: true}  ❌ system error
  JSON.stringify(...)   ❌ system error

  结论：download_params 必须是对象类型，传空对象可用，
  但传任何实际内容都会导致参数校验失败。

十、结论
────────────────────────────────────────────────────────────────

  1. 水印性质："豆包AI生成"水印是在视频生成阶段烧录进像素的，
     非播放器叠加层，非半透明 CSS 覆盖。

  2. CDN 机制：ByteDance CDN（douyinvod.com）和阿里云 OSS
     （doubao.com）均采用内容寻址，URL query 参数仅用于日志统计，
     无法通过修改参数获取不同内容。

  3. API 设计：豆包后端只对外暴露带水印的视频内容。所有公开 API
     端点（共测试 14 个）均返回相同的带水印视频。

  4. watermark_download API 虽然名字暗示去水印功能，但实际仅
     返回传入的 URL 原文，task_id 为空，未执行任何去水印操作。
     该 API 可能需要登录态 + 特定 scene 参数才能触发实际任务，
     但未登录状态下无法验证。

  5. 无水印版本推测：无水印视频可能仅存储在豆包内部创作管理
     系统中，需要创作者登录账号后通过创作后台导出。对外分享的
     视频始终是带水印版本。

  最终判定：在未登录豆包账号的情况下，无法通过公开 API 或
  URL 参数获取该视频的无水印版本。
